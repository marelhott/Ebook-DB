# Priority Pivot

## Confirmed Product Pillars

1. Library OS
2. Wireless Delivery
3. Translation Layer
4. AI Author and Book Intelligence
5. AI Recommendation Engine

## Practical Meaning

- The product does not win by becoming a desktop reader.
- The product wins by becoming the cleanest system for owning, enriching, sending, translating, and understanding books.
- Web reading remains a fallback utility, not the center of the roadmap.

## What Moves Down

- desktop reading UX polish
- reader-heavy roadmap decisions
- broad discovery homepage
- social features

## What Moves Up

- delivery jobs and device registry
- translation jobs and translated assets
- author pages with real intelligence
- recommendation rails with reasons

## Implementation Rule

Every major feature must answer at least one of these:

- Does it make the owned library cleaner?
- Does it make sending a book to a device easier?
- Does it make translation practical?
- Does it deepen author or book understanding?
- Does it improve recommendation quality?

If not, it is secondary.
